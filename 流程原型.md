## 2. 用户注册与会员

```mermaid
flowchart TD
    A[打开 APP] --> B{是否已登录}
    B -->|否| C[注册/登录]
    B -->|是| D[进入首页]

    C --> E{是否填写邀请码}
    E -->|是| F[绑定邀请关系]
    E -->|否| G[直接注册]
    F --> H[注册成功]
    G --> H

    H --> I[自动赠送 1 个月会员]
    I --> J[进入首页]
    J --> K[会员 30 天倒计时开始]

    K --> L{会员是否到期}
    L -->|未到期| M[正常使用]
    L -->|已到期| N[会员失效]
    N --> O[现金购买需重新开通]
    N --> P[纯积分兑换仍可用]
```

### 关键规则

| 节点 | 规则 |
|------|------|
| 新用户注册 | 当天起赠送 1 个月会员 |
| 邀请码 | 可选填，填写后与邀请人建立绑定 |
| 会员到期 | 需重新开通才能使用现金购买 |
| 积分保留 | 到期前累计积分保留（使用细则待确认） |

## 3. 购物下单主流程

```mermaid
flowchart TD
    A[浏览商品] --> B[商品详情]
    B --> C{操作}
    C -->|加入购物车| D[购物车]
    C -->|立即购买| E[订单确认页]
    D --> F[勾选商品去结算]
    F --> E

    E --> G{校验每日限额}
    G -->|超过 500 元| H[提示超出今日购买上限]
    G -->|未超限| I{选择支付方式}

    I --> J[支付分支见流程 4]
    J --> K[计算配送费]
    K --> L[选择自提点]
    L --> M[确认下单]
    M --> N[生成订单]
    N --> O[生成核销码]
    O --> P[订单状态: 待收货]

    P --> Q{后台是否启用配送提示}
    Q -->|是| R[展示次日配送时间与自提点]
    Q -->|否| S[仅展示自提点地址]
    R --> T[推送取货通知]
    S --> T
```

### 页面串联

```
首页/分类/搜索 → 商品详情 → [购物车] → 订单确认页 → 支付 → 支付结果 → 订单详情
                                      ↘ 立即购买 ↗
```

### 配送费计算

| 订单金额 | 默认配送费 |
|----------|------------|
| ≤ 100 元 | 3 元 |
| 101–200 元 | 6 元 |
| 后台可配置 | 可关闭或自定义档位 |

## 4. 支付方式分支

```mermaid
flowchart TD
    A[订单确认页] --> B{支付方式}

    B -->|纯积分| C{积分是否足够}
    C -->|是| D[扣减积分]
    C -->|否| E[提示积分不足]
    E --> F{是否改用积分+现金}
    F -->|是| G[进入混合支付分支]
    F -->|否| H[取消或继续赚积分]

    D --> I[下单成功]
    G --> J{是否会员}
    J -->|否| K[引导开通会员]
    K --> L[会员开通页]
    L --> M[开通成功]
    M --> N[积分扣减 + 现金补差支付]
    J -->|是| N
    N --> I

    B -->|现金支付| O{是否会员}
    O -->|否| P[弹窗提醒开通会员]
    P --> Q[跳转微信/支付宝开通]
    Q --> R[开通成功]
    R --> S[现金支付]
    O -->|是| S
    S --> I

    B -->|积分+现金| G

    I --> T[生成订单与核销码]
```

### 支付规则矩阵

| 支付方式 | 需要会员 | 价格 | 积分不足时 |
|----------|----------|------|------------|
| 纯积分 | 否 | 会员价 | 不可下单，可引导混合支付 |
| 现金 | 是 | 会员价 | — |
| 积分+现金 | 是 | 会员价 | 积分部分 + 现金补差 |

## 5. 自提点选择

```mermaid
flowchart TD
    A[订单确认页/收货地址] --> B[获取用户定位]
    B --> C[匹配附近自提站点]
    C --> D[展示可选站点列表]
    D --> E{用户选择}
    E --> F[选中自提点]
    F --> G[保存为取货站点]

    H[注意: 不支持手动输入任意地址] -.-> D

    G --> I[下单时绑定该自提点]
    I --> J[订单页展示自提点详细地址]
    J --> K[消息通知取货地点]
```

### 交互约束

- 类似快递站模式：从系统预设站点中选择
- 根据定位推荐，如「师大」→「师大站点」
- 可绑定常用地址，但取货仍以自提点为准

| 待收货 | 含待支付、配送中、待核销 |

## 7. 代理核销流程

```mermaid
sequenceDiagram
    participant U as 普通用户
    participant App as APP
    participant S as 服务端
    participant A as 代理用户

    U->>App: 下单支付成功
    App->>S: 创建订单
    S->>App: 返回核销码
    App->>U: 展示核销码(订单详情)

    U->>A: 到自提点出示核销码

    A->>App: 打开扫一扫
    App->>S: 校验代理身份
    alt 非代理
        S->>App: 返回错误
        App->>A: 提示「您不是代理」
    else 是代理
        A->>App: 扫描核销码
        App->>S: 提交核销请求
        S->>S: 校验码有效性(一次性/时效)
        alt 核销成功
            S->>App: 核销成功
            App->>A: 显示核销完成
            App->>U: 订单状态变为已收货
            U->>A: 提货离开
        else 已核销/无效
            S->>App: 返回失败原因
            App->>A: 提示无法核销
        end
    end
```

### 角色行为

| 角色 | 入口 | 行为 |
|------|------|------|
| 普通用户 | 订单详情 | 查看/展示核销码 |
| 代理用户 | 我的 → 扫一扫 | 扫码核销 |
| 非代理 | 扫一扫 | 提示「您不是代理」 |

## 8. 赚积分流程

```mermaid
flowchart TD
    A[赚积分 Tab / 首页快捷入口] --> B{获取方式}

    B --> C[签到]
    C --> C1[签到页]
    C1 --> C2{今日是否已签}
    C2 -->|否| C3[获得积分]
    C2 -->|是| C4[提示已签到]
    C3 --> C5[写入积分明细]

    B --> D[做任务]
    D --> D1[任务列表 随机3-8条]
    D1 --> D2[用户完成任务]
    D2 --> D3[悬赏平台审核]
    D3 --> D4{审核结果}
    D4 -->|通过| D5[本系统同步通过]
    D5 --> D6[积分入账]
    D4 -->|拒绝| D7[任务失败无奖励]
    D6 --> C5

    B --> E[邀请好友]
    E --> E1[我的邀请码页]
    E1 --> E2[分享邀请码/链接]
    E2 --> E3[好友注册绑定]
    E3 --> E4[邀请人得 3 积分]
    E4 --> C5

    D6 --> F[被邀请人任务积分 10% 提成给邀请人]
    F --> C5

    C5 --> G[计入当月榜单积分]
    G --> H[当月实时排名更新]
```

## 8.1 积分排行榜月度结算

```mermaid
flowchart TD
    A[用户获得积分] --> B[计入当月累计积分]
    B --> C[APP 展示当月实时排名]
    C --> D{是否每月 1 日 00:00}
    D -->|否| E[继续累计当月积分]
    E --> B
    D -->|是| F[锁定上月榜单]
    F --> G[按名次段发放月度奖励]
    G --> H[写入获奖记录与积分明细]
    H --> I[当月积分计数清零]
    I --> J[开启新一期月度榜单]
    J --> B
```

### 周期说明

| 节点 | 规则 |
|------|------|
| 统计口径 | 仅统计 **当月入账积分** |
| 结算时间 | 每月 1 日 00:00 |
| 奖励频率 | **每月一次**，对上一自然月定榜发奖 |
| 重置 | 结算后新一期从 0 开始重新累计 |

## 11. 退货退款流程

```mermaid
flowchart TD
    A[已收货订单] --> B[申请退货]
    B --> C{退款类型}
    C -->|全额退款| D[选择全部商品]
```

## 13. 后台任务审核对接

```mermaid
sequenceDiagram
    participant U as 用户
    participant App as TGG Shop APP
    participant TGG as TGG 后台
    participant RW as 悬赏任务平台

    U->>App: 完成任务
    App->>RW: 提交任务结果
    RW->>RW: 平台审核
    RW->>TGG: 审核结果回调/Webhook
    TGG->>TGG: 同步更新任务状态
    alt 审核通过
        TGG->>TGG: 计算积分奖励
        TGG->>App: 推送积分到账
        App->>U: 积分明细更新
        TGG->>TGG: 计算邀请人 10% 提成
    else 审核拒绝
        TGG->>App: 任务状态失败
        App->>U: 无积分奖励
    end
```
